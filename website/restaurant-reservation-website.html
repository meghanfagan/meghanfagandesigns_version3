Reservation Website

Created For

"User-Centered Design" course at Rasmussen College â€“ Summer Quarter (2015)

Goal

My goal for this UX study is to observe individuals using an existing restaurant reservation website. I will use my observations to plan out my own fictitious restaurant reservation website that is clear, usable, and useful.

Materials Used

  • Adobe Photoshop
  • Pencil Project to create the wireframes
  • Creately to create the site map

Introduction

Before doing anything else, I conducted user observations. These observations involved watching three individuals use a restaurant reservation website they had never visited before. These observations helped me understand what the users enjoyed and found useful about the website, and also what was frustrating and hard to use.

Assets

I used the observations to create user personas, flows, and scenarios for three user types. These materials made it easier to connect with potential users and keep their needs and goals in mind. The scenarios helped me think about how to make the navigation and content better. The user flows helped me remember that users' backgrounds, experiences, and goals influence how they approach, understand, and use websites.
I used one of the persona user flows and scenarios to create a site map. I included five pages and a search bar on each page to provide users with different options. These options allow users to easily find what they are looking for and accomplish their goals throughout the website.

Final Result

The wireframes address users' pain points. For example, each web page includes a search bar at the top to make information easy to locate. The wireframes also include frequently used functionality (such as the Make a Reservation section) on the homepage, use clear navigation labels, and use white space and hierarchy to make information easy to find and read.

One issue I had with this project was determining how to organize the content in a way that fulfilled users' goals without being confusing or looking disorganized. To resolve this issue, I included many features, such as the Make a Reservation section, on multiple pages. This decision guaranteed that this functionality could be easily located and used throughout the website.
